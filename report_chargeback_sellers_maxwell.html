O que deve ser adicionado à tool — Priorização

Sprint 1 — Maior impacto Campos provavelmente já na API interna de CBK
chargeback_status
Estado atual da disputa: awaiting_seller_response, under_review, resolved_seller_win, resolved_buyer_win — resolve ~690 convs/mês diretamente e melhora CSAT em todas as demais.
amount_at_risk
Valor em disputa retido. Campo hoje retorna string vazia. Impacta ~1.900 convs/mês sobre "fundos retidos".
response_deadline
Data limite para o seller enviar evidências. Resolve a dúvida "até quando tenho que responder?" que aparece em ~561 convs.
Sprint 2 Informação de documentação e motivo
dispute_reason
Motivo declarado pelo comprador (fraude, não recebeu, não era o esperado). Permite Maxwell orientar o seller sobre como responder especificamente.
required_documents
Lista de tipos de evidência aceitos para o motivo específico. Elimina a dúvida "o que preciso enviar?"
document_rejection_reason
Quando o seller submeteu provas e foram recusadas, por quê. Conversas explícitas sobre "meu vídeo foi rejeitado" hoje vão 100% para humano.
fund_release_date
Previsão de liberação dos fundos após resolução da disputa. Complementa amount_at_risk para o cluster "fundos retidos".
✅
Pergunta-chave para o Rodrigo/Glo: a API interna de CBK já expõe chargeback_status, amount e response_deadline? Se sim, Sprint 1 é só desenvolvimento no MCP — sem dependência de API. Se não, há um gap de API também que precisa ser endereçado antes.
🔧
Gap adicional — Routing (independente dos campos): mesmo com os novos campos, a tool só é chamada em ~20% das conversas. É preciso investigar por que o Maxwell não a chama nas outras 80%. Pode ser um problema de prompt/descrição da tool no MCP ou critério de seleção do Maxwell.
